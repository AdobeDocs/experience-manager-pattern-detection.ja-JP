# Adobe Experience Manager Screensのドキュメントへの貢献

Adobe Experience Manager Screensのドキュメントにご協力いただきありがとうございます。 Adobeは、読者コミュニティ全体およびAdobeの他の従業員によるコンテンツの投稿を歓迎します。

このファイルは、どのように貢献できるかを説明し、貢献を行う際に従うべきガイドラインを指摘します。

## AdobeオープンSource行動規範

このプロジェクトでは、[Adobe オープン Source行動規範](code-of-conduct.md)を採用しています。 AEMのドキュメントに投稿することで、この行動規範を遵守することに同意したものとみなされます。 許容できない動作を[Grp-opensourceoffice@adobe.com](mailto:Grp-opensourceoffice@adobe.com)に報告してください。

## 貢献のためのガイドライン

Adobeは、AEM ドキュメントを改善するためのあらゆる貢献を歓迎しますが、可能な限り特定のガイドラインと標準に従って貢献してください。 詳しくは、[AEM ドキュメントへの投稿に関するガイドライン &#x200B;](guidelines.md)を参照してください。

## 貢献の方法

Adobeでは、できるだけ簡単に投稿できるようにしたいと考えています。 次の2つの方法で貢献することができます。

* [イシューの作成](#create-an-issue)
* [プルリクエストの作成](#create-a-pull-request)

どの方法を選択するかは、自分でGitHubのドキュメントを編集する快適さによって異なります。

>[!NOTE]
>
>送信した寄付は、[Adobe利用条件](https://www.adobe.com/jp/legal/terms.html)の対象となります。

### イシューの作成 {#create-an-issue}

現在のドキュメントに関する提案、観察、または何かが記述される方法に関する質問はありますか？ 次にイシューを作成します。 また、貢献すべき点があるものの、ドキュメントの編集やプルリクエストを自身で行うことに抵抗がある場合にも、問題は非常に優れています。

イシューを使用すると、改善のアイデアを簡単にドキュメントに記載でき、AEM Docs チームメンバーがレビューできます。 任意のページの右側のナビゲーションパネルの上部にある「**イシューを記録**」をクリックして、イシューを作成します。

### プルリクエストの作成 {#create-a-pull-request}

問題が発生し、その解決方法を正確に把握している場合は、プルリクエストの作成を検討してください。 プルリクエストを使用すると、ドキュメントを独自に編集できます。 AEM Docsのチームメンバーがインクルージョンのために編集内容を確認します。 任意のページの右側のナビゲーションパネルの上部にある&#x200B;**このページを編集**&#x200B;をクリックして、提案された変更を行い、プルリクエストを作成します。

すべてのプルリクエストには、署名された[&#x200B; コントリビューターライセンス契約またはCLA](https://opensource.adobe.com/cla.html)が必要です。 CLAは、プロジェクトの一部としてコントリビューションを再配布するための権限をAdobeに付与します。 Adobe CLAは1回のみ送信する必要があるので、以前に送信したことがある場合は送信してください。

### Adobe社員の貢献

Adobeの社員でコントリビューションがある場合は、企業Gitで`AdobeDocs`組織を使用します。

Adobeは、パブリックリポジトリ上で行われた投稿に対して満足し、投稿をレビューします。 しかし、Adobeの担当者からのプルリクエストは処理が簡単で、企業リポジトリでより迅速に承認できます。 企業情報が企業リポジトリでのみ議論されるようにします。
